WORKSPACE
WEBAPP

THE PROBLEM what I saw

This webapp was the thesis project for my Master's of Science in Software Development at the University of Glasgow. The timeline to complete it was 12 weeks.

The COVID-19 Pandemic led to a major rise in the number of people working from home. Now that pandemic restrictions have been lifted, working remotely no longer has to mean working from home exclusively. The goal of this project was to create a web application that helps remote workers find great places to work anywhere they are.

A number of existing websites and apps exist to solve this problem. However, many have a limited scope and others focused exclusively on paid venues like coworking spaces. I wanted to create a project that made it convenient for people to find great places to work that were either free or nominally priced, such as libraries and cafes.

THE SOLUTION what I did

I built a web application using HTML5, CSS3, React, Node.js, and npm. It uses three of Google's APIs to help people find potential places to work: Autocomplete, Maps, and Places. It consists of two major components: a server built with Node.js and a client made with HTML5, CSS3, and React. Users can search for places to work in any location available on Google Maps. The Google Autocomplete API stops users from entering erroneous data or locations that are not available on Google Maps.

When a user searches for a place to work, they must type in and select a location from the drop down menu. Then, they are directed to a page that includes a customized Google Map with markers on the potential places to work near them. This page also includes a list view so users can view important information about individual locations they are considering visiting. Some examples of this include whether the place is currently open, its distance from the user's location, and Google's rating.

DUELYST
GAME

THE PROBLEM project scope

This game was a semester-long group project for my Master's of Science in Software Development at the University of Glasgow.

Our five-person team was provided with files that included the front end of a role-playing card game. We were tasked with building the back-end logic in Java. The game involved a human player playing against the computer, so we also had to write code for the computer to assess certain factors about the game's current state to make the best decisions.

THE SOLUTION what we did

Our team worked well together and we were able to complete the project within the deadline. Some of us took on extra roles, myself included. I was the team's Scrum Master, which entailed me co-leading meetings and working to keep the team on track with their assigned tasks.

I also worked on testing our code. I learned how to write tests in JUnit over the course of the project and wrote tests for certain functionalities. Some of these include the board rendering properly, the turn passing from the human player to the computer, and the numerical IDs for the cards matching with the correct names and abilities.

GLENCOE
WEBSITE

I made a visitor's guide to Glencoe, Scotland that includes information about the hikes that can be completed there. The site also includes information about how to reach it, nearby attractions, and an image gallery. The pages are responsive to different screen sizes. I built it primarily with HTML5 and CSS3.

GALAXY
3JS MODEL

I wanted to learn more about how to make 3D objects using Three.js. By going through the Three.js documentation and watching tutorials, I was able to make this 3d model of a galaxy. It includes larger rotating planets that use textures for a more realistic effect. It also includes stars and smaller planets whose postions are randomly generated every time the page is refreshed.
